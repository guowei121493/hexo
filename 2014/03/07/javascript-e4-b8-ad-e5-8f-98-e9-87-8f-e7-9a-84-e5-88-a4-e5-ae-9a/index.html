javascript 中变量的判定

发表于 2014-03-07   |   分类于 js   |  

在javascript中由于javascript是弱变量，如果编码有问题，或者检测不够严谨经常会出现变量转变导致出错。尤其是当遇到null和undefined情况，如果组件用到过多地方很可能由于没有变量定义的缘故而导致报错，解决的办法就是在之前进行变量的判断

1.判定null和undefine

一般可以利用if(var)如果其中变量不为空的话就会执行下方的语句或者 ceshi && ceshi()这种形式
也就是说null和undefined是不进行执行代码的，只有有此值的时候才进行。
这是一种粗糙的做法。如果需要强制判定 null还是undefined
可以用typeof函数 不过typeof只能判定六种常用值分别是

  • string
  • number
  • boolean
  • object
  • function
  • undefined

2.字符串的判定

字符串是个很好的类型，判定也非常容易
其他字符也可以转换为字符串有一个toString()的属性
如果不能转换的时候就会出现[Object type]的内容
经常可以看到也可以用来检测类型，
而且可以判断除了基础的六种类型外的
Date Regexp Math Error属性
但是对于有toString的函数的内容就无法判定
比如(new Date()).toString();
判定的是UTC的时间戳。
所以正确的做法应该是这样的

var _toString = Object.prototype.toString();
_toString.apply(new Date());
这样便可以检测出来除了六种基本类型的其他类型 
function typeOf(){
  var _toString = Object.prototype.toString();
  var _type = {
          "undefined":"undefined",
          "number":"number",
          "boolean":"boolean",
          "string":"string",
          "[object Function]":"function",
          "[object RegExp]":"regext",
          "[object Array]":"array",
          "[object Date]":"date",
          "[object Error]":"error",
          "[object Meth]":"math"
  }
  return _type[typeof o] || _type[typeof _toString.apply(o)] || (o?"object":"null");
}
